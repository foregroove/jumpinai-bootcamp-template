# 経費CSV変換ツール

> Jumpin' AI Bootcamp ハンズオン題材：レシート画像から経費CSVを生成するツール

## プロジェクト概要

レシート画像（JPEG/PNG）を Claude Vision API で読み取り、日付・店名・金額・勘定科目を抽出して CSV ファイルとして出力する Python スクリプト。

## 技術スタック（固定）

このプロジェクトでは以下のスタックを使う。**勝手に他のものを使わないこと。**

| 項目 | 値 |
|------|---|
| 言語 | Python 3.11+（3.12 / 3.13 / 3.14 も可） |
| AI API | Anthropic Claude API（`claude-sonnet-4-6` / Vision対応） |
| 画像処理 | Pillow |
| データ処理 | pandas |
| 環境変数 | python-dotenv |
| ロギング | logging（標準ライブラリ） |

## 必須ライブラリ（バージョン固定）

```txt
anthropic>=0.40.0,<0.50.0
pandas>=2.2.0,<3.0.0
Pillow>=10.4.0,<12.0.0

```

新しいライブラリを追加するときは必ずユーザーに確認すること。

## 経費科目（5分類・固定）

レシートを以下の5分類のいずれかに振り分ける。**この5つ以外を勝手に追加しないこと。**

### 会議費

社内・社外の打合せに関連する飲食代・茶菓子代。**自分1人の食事は対象外**。

- カフェ・喫茶店・ファストフード（スターバックス／ドトール／マクドナルド等）で複数名利用が示唆されるもの
- 取引先・社内メンバーとの飲食店利用（居酒屋・レストラン）
- 打合せ用の手土産・差し入れ・会議用茶菓子（和菓子店・洋菓子店）
- 会議室レンタル・コワーキングスペースのドロップイン利用料
- 出前・デリバリー（複数名分の量・複数の弁当が含まれる場合）

**グレーゾーン判定**：
- スターバックス1杯のみ → 会議費（打合せの可能性が高い前提）
- スーパー・コンビニで弁当1個 → **雑費**（自分の食事として扱う）
- スーパーで茶菓子（個包装の菓子・お茶）大量購入 → 会議費

### 消耗品費

業務で使う文房具・小物備品・PC周辺の消耗品。**食料品は含めない**。

- 文具（ノート・ボールペン・付箋・ファイル・テープ類）
- PC・オフィス小物（USBメモリ・マウス・ケーブル・電池・プリンタ用紙）
- 100円ショップでの雑貨購入（キャンドゥ・ダイソー・セリア）
- 清掃用品・トイレットペーパー等のオフィス備品
- 印刷費（コンビニコピー・キンコーズ）

**グレーゾーン判定**：
- 100円ショップで食品（飲料・お菓子）のみ → **会議費**（茶菓子扱い）または **雑費**
- 100円ショップで文具と食品が混在 → 主たる用途で判定、判別不能なら **雑費**
- スーパーでの食料品・日用品 → **雑費**（消耗品費にはしない）

### 旅費交通費

業務上の移動・出張に関わる交通費・宿泊費。

- 鉄道（JR・私鉄・地下鉄）の切符・IC チャージ・回数券
- タクシー・ハイヤー・配車サービス（GO・Uber 等）
- バス・高速バス・空港リムジン
- ETC・ガソリン代・駐車場代・コインパーキング
- 出張時の宿泊費（ホテル・ビジネスホテル）

**グレーゾーン判定**：
- 高速道路 SA/PA での飲食 → **会議費** または **雑費**（移動中の食事は雑費）
- レンタカー → 旅費交通費
- 駐車場（自社近辺の月極でない一時利用） → 旅費交通費

### 新聞図書費

業務に関連する書籍・新聞・雑誌・電子書籍購読料。

- 書店（紀伊國屋・丸善・ジュンク堂・蔦屋）での書籍購入
- Amazon・楽天ブックスでの書籍・電子書籍
- 新聞購読料（日経・朝日・読売）・業界紙
- 雑誌・専門誌・有料コラム（note の有料記事を含む）
- オンライン辞書・データベース利用料（業務関連のもの）

**グレーゾーン判定**：
- Kindle Unlimited 等の月額サブスク → 新聞図書費
- 書店で文具を一緒に購入 → 主たる金額の高い方で判定
- 漫画・小説（業務関連性が薄い） → **雑費**

### 雑費

上記4分類のいずれにも明確に当てはまらないもの。**迷ったら雑費にせず、まず4分類のどれかに該当しないか再検討する**。

- 自分1人の食事（弁当・コンビニ食・スーパーの惣菜）
- スーパー・ドラッグストアでの食料品・日用品購入
- 銀行手数料・振込手数料
- クリーニング代・名刺発注（業務用）
- 慶弔費・少額の交際費（会議費に分類できない場合）

**グレーゾーン判定の最終ルール**：
1. 用途が「打合せ」と明らか → **会議費**
2. 用途が「業務小物・文具」と明らか → **消耗品費**
3. 用途が「移動・出張」と明らか → **旅費交通費**
4. 用途が「書籍・購読」と明らか → **新聞図書費**
5. 上記いずれでもない／判別不能 → **雑費**

### 判定の優先順位（重要）

複数科目に該当しうる場合、以下の順で優先する：
1. **店名・業種から明確に推定できる科目**（例：書店 → 新聞図書費）
2. **品目の主たる用途**（例：100円ショップで茶菓子のみ → 会議費）
3. **判別不能なら雑費**（無理に他科目に寄せない）

## 出力CSV仕様（固定）

- 列順：`日付,店名,金額,勘定科目`（この順序を変えないこと）
- エンコーディング：UTF-8（BOM **なし**）
- 区切り：カンマ
- ヘッダ行：あり
- 日付フォーマット：`YYYY-MM-DD`
- 金額：整数（円、税込）

## 実装スタイル

- **エラーハンドリング**：try/except パターンで明示的に書く。silent fail させない
- **ロギング**：`print()` 禁止。`logging.info()` `logging.error()` を使う
- **型ヒント**：関数定義には型ヒントを付ける（PEP 484準拠）
- **docstring**：公開関数にはdocstringを書く（Google スタイル）
- **設定値**：ハードコーディング禁止。`.env` か定数モジュールに集約

## やってはいけないこと

- API キーをコードに直接書く（必ず `.env` 経由）
- `print()` でログ出力する（`logging` を使う）
- 5分類以外の経費科目を勝手に追加する
- ライブラリを勝手にアップグレード／追加する
- requirements.txt を更新せずに `pip install`する

## よく使うコマンド

# 実行
